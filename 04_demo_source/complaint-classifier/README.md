# VOC 智能分类与优先级评估

基于服务运营场景构建的客诉/VOC 智能分析 Demo。系统将用户问题文本转化为可运营的分类、情绪、优先级、处理建议和批量异常信号，用于模拟智能客服、服务分流和风险前置识别。

## 在线演示

https://complaint-classifier-crxmkw4rzhybwb5ksusx2d.streamlit.app

## 业务背景

在真实服务团队中，一线人员需要快速判断用户问题属于什么类型、风险等级有多高、是否需要升级处理。这个 Demo 对应“从人工经验判断到规则/AI 辅助分流”的过程，展示如何把服务 SOP 中的判断逻辑转化为可运行的产品能力。

## 核心功能

- VOC 自动分类：退款、物流、商品质量、服务态度、其他等类别。
- 优先级评估：P0 紧急、P1 重要、P2 普通。
- 情绪识别：愤怒、焦虑、平静等基础情绪判断。
- 处理建议：生成可执行的话术和操作指引。
- 批量异常检测：识别相似客诉聚集，提示潜在批量风险。
- 可视化看板：分类、优先级、情绪、交叉分布等图表。

## 引擎设计

默认规则引擎无需 API Key，可直接演示核心流程。接入 DeepSeek、Gemini、Groq 或本地 Ollama 后，可切换为 AI 增强分析。

## 技术栈

- Python
- Streamlit
- pandas
- Plotly
- OpenAI-compatible SDK

## 本地运行

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 数据说明

仓库中的样例数据为模拟数据，仅用于功能演示，不包含真实用户隐私。

## 面试展示重点

这个项目展示了三项能力：

- 将服务 SOP 中的人工判断规则产品化。
- 设计“规则引擎 + AI 增强”的可降级方案，避免 Demo 依赖 API Key。
- 从单条客诉分析扩展到批量异常识别，体现服务风险前置治理思路。
