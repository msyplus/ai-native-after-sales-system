# 服务事件智能摘要

面向客服工单、对话记录和操作日志的智能摘要 Demo。系统将长文本服务记录压缩为结构化摘要，模拟 AI 自动总结在工单录入、事件复盘和交接协同中的应用。

## 在线演示

https://summary-system-4.streamlit.app

## 业务背景

在复杂服务场景中，一线人员往往需要从聊天记录、订单日志和历史工单中整理事件经过。真正耗时的不是“写字”，而是把分散信息拼成后续角色可以理解的结构化事实。这个 Demo 对应 AI 自动总结项目，复现从 Prompt 约束到结构化输出的设计思路。

## 核心功能

- 多源输入：支持对话、日志、备注等不同文本来源。
- 结构化摘要：输出事件类型、用户诉求、关键节点、已处理动作、待跟进事项。
- 关键词与实体提取：识别时间、金额、关键对象和风险词。
- 双引擎对比：默认规则摘要 + AI 增强摘要。
- 人工评分反馈：模拟摘要质量回流和 Badcase 记录。

## 引擎设计

默认关键词/模板摘要无需 API Key，可直接展示核心流程。接入 DeepSeek、Gemini、Groq 或本地 Ollama 后，可体验更接近真实 LLM 的摘要效果。

## 技术栈

## 本地运行

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 数据说明

内置样例均为模拟数据，仅用于演示摘要、评分和报告逻辑。

## 面试展示重点

这个项目是作品集中最贴近 AI 产品运营主线的一项，适合展示：

- 如何从一线 SOP 中识别适合 LLM 的高频、低风险场景。
- 如何通过固定模板、必填字段、字数约束和 Badcase 回流提升摘要质量。
- 如何把“模型生成文本”嵌入工单工作流，而不是制造新的工具负担。
